这里写开场。建议用一两段快速说明：

- 这篇文章为什么值得读
- 你想解决什么问题
- 你的核心判断是什么

## 背景

交代问题是怎么来的、发生在什么场景里、为什么值得展开。

## 核心观点

把最重要的判断先讲清楚。可以是一个方法论、一次复盘结论，或者一个你反复验证过的经验。

## 展开分析

这里可以细讲你的过程、对比、拆解、案例或推导。

### 小节标题

如果内容层次比较多，可以继续分二级、三级标题。

你现在可以直接使用常见 Markdown 能力，例如：

- 无序列表
- 有序列表
- 表格
- 引用
- 代码块
- 图片

> 这里可以放一句值得强调的话。

```ts
export function example() {
  return "hello blog";
}
```

## 结论

最后收束全文，回答两个问题：

- 这篇文章最后得出了什么结论
- 读者读完之后可以带走什么

## 延伸

如果你想给读者一个后续动作，也可以在最后加：

- 推荐阅读
- 相关案例
- 下一篇会写什么
