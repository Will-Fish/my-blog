## 一、项目背景与立项说明

### 1.1 背景

传统简历是静态文本，信息密度高但人格化表达不足，难以充分传递候选人的综合能力与个性。随着个人品牌意识的提升，越来越多的求职者选择通过个人网站作为简历的延伸载体。

与此同时，本人具备多项技能——不仅有产品经理相关的思考与输出能力，还有摄影、视频创作等创意技能。这些能力在传统简历中无法得到充分展示。

因此，需要构建一个**能同时承载文字内容输出与视觉媒体作品展示**的个人网站，形成完整的个人品牌载体。

### 1.2 项目价值

| 价值维度 | 说明 |
|---------|------|
| 求职展示 | 让面试官快速了解我是谁、做过什么、为什么适合产品经理岗位 |
| 个人品牌 | 持续输出产品思考、项目复盘、摄影作品，建立个人影响力 |
| 产品练手 | 完整走一遍 PM 做产品的流程，沉淀为可讲的项目经历 |
| 多维展示 | 突破文字局限，通过图片和视频展示更立体的个人形象 |

### 1.3 项目定位

> 一个面向招聘者与同行的**个人品牌与内容展示平台**，集博客文章、产品项目案例、摄影图集、视频作品于一体，体现产品思维、表达能力与创意实力。

## 二、目标与成功标准

### 2.1 用户目标

- 面试官能在 **10 秒内**判断候选人的定位与能力匹配度
- 面试官能方便地看到产品项目案例和博客文章
- 访客能沉浸式浏览摄影图集与视频作品
- 访客能便捷获取联系方式或下载简历

### 2.2 业务目标

- 网站成功上线并可公网访问
- 支持博客文章、项目展示、摄影图集、视频作品四大内容模块
- 成为简历中的附加入口链接，提升求职转化率
- 作为完整 PM 项目经历，可在面试中完整讲述需求→方案→取舍→迭代过程

### 2.3 成功标准

| 指标 | 目标值 | 验收时间 |
|------|-------|---------|
| 网站上线可访问 | 100% | v1.0 发布时 |
| 首页首屏核心信息传达完整度 | 面试官 10 秒内可了解候选人定位 | 用户测试验证 |
| 内容上线数量（博客） | ≥5 篇 | 上线后 2 周内 |
| 摄影作品上线数量 | ≥20 张图片 + ≥2 个视频 | 上线后 2 周内 |
| 页面移动端适配 | 所有页面在主流移动设备正常展示 | 测试验收时 |

## 三、用户分析

### 3.1 用户画像

#### 用户 A：招聘者 / 面试官（核心用户）

| 属性 | 描述 |
|------|------|
| 身份 | HR、用人部门负责人、产品总监 |
| 使用场景 | 从简历链接点击进入，用于评估候选人 |
| 核心关注点 | 候选人背景、求职方向、项目经历、产品能力 |
| 行为特征 | 浏览时间短（1-5 分钟），目标明确，倾向于快速扫描 |
| 核心需求 | 快速了解候选人、快速找到项目案例、获取联系方式 |

#### 用户 B：同行 / 同学 / 朋友（次要用户）

| 属性 | 描述 |
|------|------|
| 身份 | 产品同行、设计师、开发者、创作者 |
| 使用场景 | 通过社交平台分享链接进入 |
| 核心关注点 | 项目拆解思路、博客内容、摄影视频作品 |
| 行为特征 | 浏览时间较长，更注重内容深度，可能多次访问 |
| 核心需求 | 看到高质量内容输出、获得灵感与参考 |

#### 用户 C：站长本人（管理用户）

| 属性 | 描述 |
|------|------|
| 身份 | 网站所有者 |
| 使用场景 | 日常更新文章、上传作品、维护内容 |
| 核心关注点 | 内容管理便捷性、更新效率 |
| 行为特征 | 高频操作，需要低门槛的内容更新方式 |
| 核心需求 | 便捷发布文章、上传图片/视频、管理已有内容 |

### 3.2 用户场景与需求映射

| 用户 | 场景 | 需求 | 对应功能 |
|------|------|------|---------|
| 面试官 | 点开简历链接 | 快速了解候选人定位 | 首页个人简介、岗位定位 |
| 面试官 | 评估产品能力 | 查看项目案例与产品思考 | 项目展示页、博客页 |
| 面试官 | 想进一步联系 | 快速获取联系方式 | 联系页、简历下载 |
| 同行 | 浏览内容 | 看到有深度的文章和项目拆解 | 博客详情页、项目详情页 |
| 同行 | 欣赏摄影作品 | 浏览图集与视频 | 摄影画廊、视频作品页 |
| 站长 | 日常维护 | 便捷更新内容 | 内容管理（Markdown/CMS） |

## 七、功能详细说明

### 7.1 首页

**页面目标**：访客 10 秒内了解我是谁、能做什么

**内容区块（从上到下）**：

| 区块 | 内容说明 | 备注 |
|------|---------|------|
| Hero 区 | 个人定位语、姓名、一句话介绍、CTA按钮（查看项目 / 联系我） | 首屏，最核心 |
| 核心能力 | 3-5 个技能标签或简短说明（产品思维、摄影、内容创作等） | 快速建立人设 |
| 精选项目 | 2 个代表项目卡片，点击进入项目详情 | 面试官核心诉求 |
| 摄影预览 | 展示 3-6 张精选摄影作品缩略图，引导进入摄影专区 | 体现多元技能 |
| 最新文章 | 最近 3 篇博客文章标题+摘要 | 展示内容持续输出 |
| 联系方式 | 邮箱 / GitHub / 简历下载 | 转化入口 |

**验收标准**：
- [ ] 首屏在 1920x1080 分辨率下完整显示 Hero 区内容
- [ ] 移动端（375px宽）布局正常，核心信息可读
- [ ] 所有 CTA 按钮点击后正确跳转
- [ ] 摄影预览图片正常加载，点击跳转摄影页

### 7.5 摄影模块（核心新增模块）

摄影模块是本站区别于普通产品经理个人网站的**差异化亮点**，需要提供高质量的视觉浏览体验。

#### 7.5.1 摄影模块入口

- 顶部导航设置「摄影」一级菜单
- 点击展开二级菜单：图片集 / 视频集
- 首页设置摄影预览区块，引导用户进入

#### 7.5.2 图片集页面

**功能说明**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 瀑布流/网格布局 | 图片以瀑布流或等比网格展示，避免裁剪破坏构图 | P0 |
| 全屏 Lightbox | 点击图片后全屏展示，支持左右切换浏览 | P0 |
| 分类筛选 | 支持按主题分类：风光/人像/城市/旅行/纪实等（v1.1） | P1 |
| 图片信息展示 | 鼠标悬浮或Lightbox中展示图片标题、拍摄地点 | P1 |
| EXIF信息 | 展示相机参数（焦距/光圈/快门/ISO）（v2.0） | P2 |
| 懒加载 | 图片进入视口时再加载，优化页面性能 | P1 |

**交互说明**：

- 鼠标悬浮图片时显示遮罩 + 图片标题
- 点击图片进入 Lightbox 全屏查看
- Lightbox 内支持：左右箭头切换、键盘方向键切换、点击遮罩/ESC关闭
- 移动端支持左右滑动切换

**验收标准**：
- [ ] 图片瀑布流布局在桌面端（1440px）正常显示
- [ ] 图片瀑布流布局在移动端（375px）正常显示（单列或双列）
- [ ] Lightbox 全屏查看功能正常
- [ ] Lightbox 内左右切换功能正常
- [ ] ESC 键和点击遮罩可关闭 Lightbox
- [ ] 图片懒加载生效（未进视口前不发起加载请求）

#### 7.5.3 视频集页面

**功能说明**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 视频卡片列表 | 以卡片展示视频封面图、标题、简介、时长 | P0 |
| 页面内视频播放 | 点击视频卡片后在当前页面内嵌播放，不跳转外部平台 | P0 |
| 视频来源支持 | 支持本地视频文件 + 嵌入B站/YouTube iframe | P0 |
| 视频分类 | 按主题分类（Vlog/延时/人文纪录等）（v1.1） | P1 |
| 视频描述 | 每个视频附带文字描述/拍摄背景说明 | P1 |

**视频播放交互**：
- 方案A（推荐）：点击视频卡片后弹出模态框（Modal），在框内嵌入播放器，点击遮罩或关闭按钮关闭
- 方案B：点击卡片后跳转视频详情页，详情页内嵌播放器

> 推荐方案A，用户无需离开列表页，浏览体验更流畅

**验收标准**：
- [ ] 视频封面图正常显示
- [ ] 点击视频卡片后视频可正常播放
- [ ] 视频播放器控件完整（播放/暂停/进度条/音量/全屏）

## 八、用户故事

### 核心用户故事

**US-01**  
作为面试官，我想在进入网站首页后 10 秒内了解候选人的求职方向和核心能力，以便快速判断是否符合我的招聘需求。

**US-02**  
作为面试官，我想浏览候选人的产品项目案例，查看其需求分析和方案设计过程，以便评估其产品思维能力。

**US-03**  
作为面试官，我想方便地下载候选人的简历 PDF 文件，以便在内部进行候选人评估时使用。

**US-04**  
作为一名访客，我想在摄影图片集页面流畅浏览高质量图片，并支持全屏查看，以便获得沉浸式的视觉欣赏体验。

**US-05**  
作为一名访客，我想在不离开当前页面的前提下直接播放摄影视频，以便保持浏览的连贯性。

**US-06**  
作为一名同行，我想按分类筛选博客文章（如产品分析、项目复盘、AI工具等），以便快速找到我感兴趣的内容。

**US-07**  
作为一名摄影爱好者，我想按主题（风光、人像、城市等）筛选图片作品，以便欣赏我最感兴趣的类型。

**US-08**  
作为站长，我想通过 Markdown 文件便捷发布博客文章，而不需要登录复杂的后台系统，以便降低内容更新的门槛。

**US-09**  
作为站长，我想通过修改配置文件或上传图片文件来更新摄影图集，以便保持作品库的持续更新。

## 九、非功能性需求

### 9.1 性能要求

| 指标 | 目标 |

## 十、技术方案说明

> 以下为产品侧的技术选型建议与取舍逻辑，具体实现由开发同学决策

### 10.1 推荐技术栈

| 层次 | 推荐方案 | 理由 |
|------|---------|------|
| 前端框架 | Next.js / Nuxt.js | SSG静态生成，SEO友好，性能好 |
| 样式方案 | Tailwind CSS | 开发效率高，响应式便捷 |
| 博客内容管理 | Markdown + Git | 轻量，无需复杂后台，版本可控 |
| 图片托管 | 阿里云 OSS + CDN | 已有服务器，扩展方便，全国访问速度快 |
| 视频托管 | B站投稿嵌入 | 无流量成本，国内访问稳定 |
| 部署 | 阿里云服务器 + Nginx | 已有基础设施，成本低 |
| 统计分析 | 百度统计 / Google Analytics | 免费，接入简单 |

### 10.2 摄影模块图片方案说明

| 方案 | 优点 | 缺点 |
|------|------|------|
| 阿里云OSS存储 | 访问速度快，可自定义CDN域名 | 有存储和流量费用 |
| 本地服务器存储 | 无额外费用 | 服务器带宽有限，并发差 |
| 七牛云/又拍云 | 有免费额度，图片处理功能强 | 免费额度有限 |

**建议**：v1.0 用本地存储或阿里云 OSS 基础版，图片压缩至 800KB 以内；v2.0 按访问量评估是否需要 CDN 加速。

### 10.3 关键技术取舍记录

| 决策点 | 选择 | 放弃 | 取舍理由 |
|--------|------|------|---------|
| 博客系统 | Markdown文件管理 | 自建后台CMS | 个人站不需要复杂后台，Markdown更轻量 |
| 视频托管 | B站嵌入iframe | 自托管视频文件 | 节省服务器带宽和存储成本 |
| 是否需要数据库 | v1.0不需要 | 接入MySQL | 静态站+配置文件已满足v1.0需求 |
| 评论系统 | Giscus（基于GitHub Issues） | 自建评论服务 | 零成本，访客可用GitHub账号评论 |
